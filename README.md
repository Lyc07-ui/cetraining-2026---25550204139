##项目简介
小型图书借阅管理系统是一个基于Python的命令行应用程序，采用面向对象编程思想设计。系统包含图书信息录入、借阅图书、归还图书、查询借阅状态等核心功能，并支持借阅记录持久化存储和按书名或作者的模糊查询。系统内置15本示例图书，数据以JSON格式保存到本地文件，程序启动时自动加载，操作后自动保存。

**如何运行项目**

1. 环境准备：确保已安装Python 3.7或更高版本。打开命令行输入`python --version`检查版本，若未安装请从python.org下载安装。

2. 保存代码：将代码完整复制到一个新建文件中，命名为`library.py`。建议存放在一个独立的项目文件夹内，如`D:\Projects\LibrarySystem\`。

3. 运行方式一（PyCharm）：打开PyCharm，选择File -> Open打开项目文件夹，双击`library.py`文件，右键点击编辑区选择Run 'library'，或直接点击右上角绿色运行按钮。

4. 运行方式二（命令行）：打开终端，切换到项目文件夹路径，执行`python library.py`回车即可启动。

5. 运行方式三（VS Code）：安装Python扩展后，打开`library.py`文件，点击右上角运行图标或按F5启动调试。

6. 首次运行：程序会自动检测同目录下是否存在`library_data.json`数据文件，若不存在则使用内置15本示例图书并创建该文件，后续所有操作数据均自动保存至此文件。

7. 注意事项：请勿删除或修改`library_data.json`文件，否则下次启动将重新加载默认示例数据，丢失之前的操作记录。如需备份，直接复制该JSON文件即可。

**功能演示说明**

1. 系统启动：程序运行后首先显示欢迎信息"欢迎使用图书借阅管理系统！已内置15本示例图书"，随后加载数据文件并打印主菜单，菜单包含8个选项，用户输入对应数字即可操作。

2. 图书信息录入：选择选项1，按提示依次输入图书编号（如B016）、书名（如"机器学习实战"）、作者（如"彼得·哈灵顿"）、库存数量（如5）。系统会自动检查编号是否已存在，若重复则提示错误并要求重新输入。录入成功后立即显示图书信息并持久化到JSON文件。

3. 借阅图书：选择选项2，系统先展示全部图书的编号、书名、作者、库存、累计借出次数列表，用户输入目标图书编号（如B001）和借阅人姓名（如"张三"）。系统校验图书存在性与库存数量，若库存大于0则执行借阅：对应图书库存减1，累计借出次数加1，生成一条包含唯一记录ID、借阅时间戳的借阅记录。若库存为0则提示"库存不足，无法借阅"。

4. 归还图书：选择选项3，系统展示所有借阅记录列表，包含记录ID、图书名、借阅人、借阅时间、状态（借阅中/已归还）。用户输入要归还的记录ID（如1），系统校验记录存在且未归还，执行归还：对应图书库存加1，记录更新归还时间戳，状态变为"已归还"。若输入已归还的记录ID则提示"该图书已归还"。

5. 查询图书借阅状态：选择选项4，输入图书编号（如B006）查看单本详情，显示图书基本信息、历史借阅记录列表、当前借阅中的人员名单。直接按回车则查看全部图书的汇总表格，包含编号、书名、作者、库存、累计借出次数。

6. 模糊查询图书：选择选项5，输入查询关键字（如"python"或"刘"），系统对书名、作者、编号三个字段进行大小写不敏感模糊匹配，返回所有符合条件的图书列表，显示编号、书名、作者、库存。若未找到则提示"未找到包含'xxx'的图书"。

7. 查看所有图书：选择选项6，以格式化表格展示全部15本图书的完整信息，与查询功能中的汇总表格式一致。

8. 查看所有借阅记录：选择选项7，以格式化表格展示所有历史借阅记录，包括已归还和借阅中的记录，按记录ID排序。

9. 退出系统：选择选项0，程序正常结束，所有数据已自动保存到JSON文件，下次启动时完整恢复。

**AI使用说明**

1. 整体架构设计：项目的三个核心类（Book、BorrowRecord、Library）的属性和方法定义由AI生成。Book类封装图书编号、书名、作者、库存、累计借出次数；BorrowRecord类封装记录ID、图书关联信息、借阅人、时间戳、归还状态；Library类作为业务逻辑层，整合图书列表和记录列表，提供完整的增删改查接口。

2. 数据持久化方案：AI建议使用JSON格式实现数据持久化，通过to_dict方法将对象转换为字典，from_dict类方法将字典还原为对象。程序启动时调用_load_data自动读取，每次数据变更后调用_save_data自动写入。该方案无需额外安装数据库，适合小型项目快速实现。

3. 模糊查询实现：AI使用Python字符串的lower()方法统一大小写，配合in操作符进行子串匹配，遍历图书列表的书名、作者、编号三个字段，返回所有包含关键字的图书。实现简单且无需依赖外部搜索引擎。

4. 交互菜单设计：AI采用while True循环配合input函数构建命令行菜单，通过if-elif结构分发用户选择到对应业务方法。菜单界面使用等号和中文字符格式化，提升可读性。

5. 问题一：初始版本未内置示例数据，用户反馈"直接帮我生成一份图书，不需要联系外部文件"。修正方式：在Library类的_init_sample_data方法中硬编码15本覆盖计算机、文学领域的示例图书，程序首次启动时自动加载，无需用户手动准备数据文件。

6. 问题二：在配套开发的词频统计工具中，PyCharm直接运行时报错"请提供至少一个输入文件路径"，因为argparse将文件参数设为必填。修正方式：将文件参数改为nargs='?'可选，并增加DEFAULT_FILE_PATH默认路径配置，当命令行未提供参数时自动使用默认值，使PyCharm直接点击运行即可工作。

7. 问题三：用户多次强调"直接给我代码""不需要讲解错误"，说明偏好结果导向的交付方式。修正方式：后续回复中减少了解释性文字和错误分析过程，直接提供完整可运行代码，将说明内容放在代码注释中，用户需要时可自行阅读。

8. 问题四：用户要求输出"纯文和数字，不要表格"，说明对格式有明确偏好。修正方式：在图书管理系统和词频统计工具的项目说明中，避免使用markdown表格，改用文字描述和列表形式呈现信息。
